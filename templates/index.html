🔍 Primary Key Detection Rules

1
Explicit PK suffix — Any column whose name ends with _PK (case-insensitive) is treated as part of the primary key. E.g. patient_PK, ORDER_PK.
2
DDL constraint — Columns declared with PRIMARY KEY inline or via a table-level PRIMARY KEY (…) clause.
3
Surrogate key pattern — Columns named exactly id or ending in _id are treated as surrogate keys. E.g. user_id, order_id.
!
Volatility score: (unique values − 1) / (total versions − 1). A score of 0 means the column never changes; 1 means it changes on every single version update.
